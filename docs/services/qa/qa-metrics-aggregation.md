# QA 指标聚合接口

> 分支：`LiXiChuanTeam/feat/qa-metrics-aggregation`
> 日期：2026-06-29
> 状态：已实现

## 修改内容

本拉取请求修正了问答指标聚合接口的统计口径，确保四个指标端点提供稳定、一致的统计数据。

1. **概览接口** — 在指标概览结构体中新增知识库数量和文档数量字段（暂为零，后续对接知识库服务）；将问答次数从消息表改由问答运行记录表统计，过滤条件为有效运行状态。
2. **趋势接口** — 将按日聚合的数据源从消息表切换为问答运行记录表，使用生成完整日期序列并通过左连接对无数据日期补零。
3. **意图分布接口** — 将统计来源从消息的意图字段改为运行记录的意图类型字段，空值归类为未知，百分比舍入保留一位小数。

涉及四个接口：

| 方法 | 路径 | 参数 |
|------|------|------|
| `GET` | `/api/v1/qa-metrics/overview` | 统计天数（默认一天） |
| `GET` | `/api/v1/qa-metrics/trend` | 统计天数（默认三十天） |
| `GET` | `/api/v1/qa-metrics/top-queries` | 统计天数（默认七天）、返回条数（默认十条，最大一百） |
| `GET` | `/api/v1/qa-metrics/intent-distribution` | 统计天数（默认七天） |

## 统计口径

| 指标 | 来源表 | 过滤条件 |
|------|--------|----------|
| 问答总次数 | 运行记录表 | 状态为已完成或运行中 |
| 今日问答数 | 运行记录表 | 状态为已完成或运行中，且开始时间为今日 |
| 平均延迟 | 运行记录表 | 完成时间在统计窗口内，且状态为已完成 |
| 今日活跃用户 | 运行记录表关联会话表 | 开始时间为今日，且状态为已完成或运行中 |
| 趋势数据点 | 运行记录表 | 按日聚合，无数据日期补零 |
| 意图分布 | 运行记录表的意图类型字段 | 空值归类为未知，一位小数舍入 |

## 趋势完整日期补零

使用生成日期序列函数生成完整日期，通过左连接对无数据日期自动补零。

## 意图分布舍入

- 分母：所有意图类型的数量总和
- 舍入：数量除以总数乘一千后四舍五入再除以十，保留一位小数
- 中文标签映射：知识问答对应知识问答、一般对话对应一般对话、报告生成对应报告生成、数据分析对应数据分析、其他对应未知

## 参数校验

| 参数 | 校验规则 |
|------|----------|
| 统计天数 | 必须为正整数，非法返回字段级错误 |
| 返回条数 | 必须在一到一百之间，超出返回字段级错误 |
